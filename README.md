## Gopher tortoise (*Gopherus polyphemus*) Genome Mapping

## Background

This project is part of a larger effort to assemble the genome of the 
gopher tortoise (*Gopherus polyphemus*) using long-read Oxford Nanopore sequencing data. A reproducible pipeline was developed to demonstrate the importance of sequencing coverage depth in reference-guided genome mapping.

## Objective

Map Nanopore reads from individual GT_7.4_B against the Bolson (Mexican) tortoise (*Gopherus flavomarginatus*) reference genome ([BioProject:PRJNA794188] (https://www.ncbi.nlm.nih.gov/bioproject/PRJNA794188/) at two different coverage depths. We are going to map 10% of the data (small dataset) versus 100% of the data (large dataset) and will compare the results.

## Scripts

| Script | What each script does |
|--------|-------------|
| `genome_mapping_utils.py` | Utility module with functions for running shell commands, validating files, extracting sample names, and parsing alignment statistics |
| `01_subsample.py` | Selects a small (13 files, ~10%) and large (135 files, 100%) subset of FASTQ files |
| `02_align_reads.py` | Aligns selected FASTQ files to the reference genome using minimap2 |
| `03_summarize_results.py` | Runs samtools stats on BAM files and generates a comparison summary |
| `04_plot_results.py` | Calculates estimated coverage depth and generates a bar chart |
| `submit_alignment.sh` | SLURM submission script for running alignment on the HPC cluster |

## How to Run

**`genome_mapping_utils.py`** — Utility module (imported by all scripts)

This is a custom Python module that is 
imported by all other scripts in the pipeline. It contains four 
reusable functions:
- `run_command()` — runs shell commands like minimap2 and samtools 
  from within Python and checks for errors
- `validate_file()` — checks that an input file exists and is not 
  empty before processing
- `extract_sample_name()` — uses regular expressions to extract the 
  sample name from a FASTQ filename
- `parse_samtools_stats()` — uses regular expressions to parse the 
  text output of samtools stats and extract key alignment metrics

By having these functions in one module, all pipeline scripts 
can import and reuse them without duplicating code. Additionally, if we make changes to any of the functions, we can do so in this module instead of changing them in each script in the pipeline.

---

**Step 1** — Select FASTQ subsets:
```bash
python3 01_subsample.py
```
This script reads the `fastq_pass` directory containing 135 gzipped 
FASTQ files from individual GT_7.4_B. The FASTQ files are the output of Nanopore sequencing. The script uses regular expressions to 
filter for `.fastq.gz` files and sort them numerically by file number. 
It then selects two subsets, a small subset of 13 files (~10% of the 
data) and a large subset of all 135 files (100% of the data), and 
writes the file paths to two text files:
- `small_subset_files.txt`
- `large_subset_files.txt`

---

**Step 2** — Align reads to the reference genome (submitted to SLURM):
```bash
sbatch submit_alignment.sh small
sbatch submit_alignment.sh large
```
`submit_alignment.sh` is a SLURM batch script that requests CPU 
resources on the HPC cluster and loads the required modules (minimap2 
and samtools). It then calls `02_align_reads.py` with either `small` 
or `large` as an argument.

`02_align_reads.py` reads the file list generated by `01_subsample.py`, 
validates the input files and the reference genome, and runs minimap2 to align the Nanopore reads to the pre-indexed 
*Gopherus flavomarginatus* reference genome. The output is piped 
directly into samtools to produce a sorted BAM file. Outputs:
- `FBE04231_small.bam`
- `FBE04231_large.bam`

---

**Step 3** — Summarize alignment results:
```bash
python3 03_summarize_results.py
```
This script runs `samtools stats` on both BAM files and uses regular 
expressions to extract key alignment metrics including total reads, 
total bases, mapped reads, mapping rate, mean read length, and error 
rate. It then writes a formatted comparison table to:
- `alignment_summary.txt`

The following metrics are extracted and compared between the two subsets:

- **Total reads** — the number of individual DNA fragments sequenced.
  Each read is one continuous piece of DNA produced by the Nanopore
  flow cell.
- **Total bases** — the total number of DNA bases (A, T, G, C) across
  all reads combined. This is the most meaningful measure of how much
  of the genome can be covered.
- **Mapped reads** — the number of reads that successfully aligned to
  the *Gopherus flavomarginatus* reference genome.
- **Mapping rate** — the percentage of reads that successfully mapped
  to the reference genome. A high mapping rate confirms that
  *Gopherus polyphemus* is closely related to the reference species.
- **Mean length** — the average length of reads in base pairs. A
  consistent mean length between subsets confirms they are
  representative samples of the same sequencing run.
- **Error rate** — the percentage of bases that do not match the
  reference genome. This includes both true biological differences
  between the individual and the reference and sequencing errors

---

**Step 4** — Plot coverage depth comparison:
```bash
conda activate biol-7180
python3 04_plot_results.py
```
The `biol-7180` conda environment must be activated before running
this script because it contains the **Matplotlib** library which is
required for generating figures. Matplotlib is not available in the
default Python environment on the cluster.

This script reads the `alignment_summary.txt` file generated by 
`03_summarize_results.py` and uses regular expressions to extract 
the total bases for each subset. It then calculates the estimated 
genome coverage depth by dividing the total bases by the *Gopherus 
flavomarginatus* genome size (~2.4 GB) and generates a bar chart 
comparing coverage depth between the two subsets. Output:
- `coverage_comparison.png`
